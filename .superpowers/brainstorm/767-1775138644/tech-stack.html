What tech stack should we use?

This shapes how the app is built and deployed

A

Pure Frontend (HTML/CSS/JS)

Single HTML file or simple static site. No backend, no database. Everything runs in the browser. Colors and collections stored in localStorage. Easy to share as a file or deploy to GitHub Pages / Netlify for free.

Pros

  • Zero setup, works offline
  • Free to host anywhere
  • No account needed

Cons

  • Data tied to one browser
  • No sharing between devices
B

React + Vite (Frontend framework)

Modern component-based app. Still no backend — localStorage or IndexedDB for persistence. Better code organization for a multi-page experience. Deploy to Vercel / Netlify.

Pros

  • Clean component architecture
  • Easy to add features later
  • Great ecosystem

Cons

  • Slightly more setup
  • Still local-only storage
C

Full Stack (Next.js + Database)

Full web app with user accounts, cloud-stored collections, and sharing features. Backend API + database (e.g., SQLite or Postgres). More infrastructure to set up.

Pros

  • Collections persist across devices
  • Can share with others
  • User accounts

Cons

  • Much more complex to build
  • Needs hosting + DB setup
